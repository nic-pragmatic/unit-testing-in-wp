Testing Methods

The two types of testing methods are Static and Dynamic

Static Testing

Informal Reviews

Walkthroughs

Code Reviews

Static testing. It is also called as Verification.

Static testing is done manually or with a set of tools. Here, the tester checks the code, design documents, requirement document and gives comments on the work document.

Informal Reviews is where a team of reviewers just checks the documents and give comments. It is non-documented in nature. PR as example

Walkthroughs is where the author explains about the software to the team and teammates can raise questions if they have any.

Code is reviewed without execution, it is checked for syntax, coding standard, and code optimization. It is also referred as white box testing.

Dynamic Testing

Unit Testing

Integration Testing

Functional Testing

Acceptance Testing

Unit Testing is testing of individual units or modules

Integration Testing is testing a group of units and modules

Functional Testing tests the system as a whole. Testing all scenarios.

Acceptance Testing ensures the product meets customer requirements

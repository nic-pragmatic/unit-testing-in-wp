REST API Basics

Important Concepts

  • REpresentational State Transfer

    Stateless operations acting on "Web resources"

  • Built on top of the HTTP protocol

    Uses URIs, HTTP verbs and HTTP status codes

  • Collections & Elements

    Every resource can be deal with as a collection, or its individual elements can be manipukated

Endpoint construction

Base URL: example.com/wp-json/wp/v2/

Collections:
GET /books - returns a list of books
POST /books - creates one or many books
PUT /books - updates one or many books
PATCH /books - partially updates one or many
DELETE /books - deletes all books

Elements:
GET /books/:id - returns a specific book based on :id parameter
POST /books/:id - creates a book with a specific :id
PUT /books/:id - updates a specific book
PATCH /books/:id - partially updates a specific book
DELETE /books/:id - deletes a specific book

Headers & Bodies

  • Headers take additional request metadata

    Authentication tokens, caching instructions, ...

  • Bodies contain the data of a POST/PATCH request

    When creating a new book entry, the body cnotains the book fields to use for that new entry.

URL arguments

  • URL arguments further qualify a given request

    i.e. stating what field to order by when fetching a collection.

  • URL arguments can also modify handling of the request

    i.e. &_embed lets WordPress embed the actual resource data of resources that were linked to.
